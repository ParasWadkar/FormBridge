# FormBridge

FormBridge is a lightweight solution that connects your custom website forms to a **Google Forms backend**. It allows users to submit data through your website while seamlessly storing responses in Google Forms — without exposing the original form UI.

---

## Overview

This project acts as a **bridge between frontend UI and Google Forms**, enabling:

- Custom-designed form interfaces on your website  
- Automatic submission of data to Google Forms  
- No need for a backend server or database setup  

It is part of a series of experiments focused on building **no-cost, scalable form integrations for early-stage startups**.

---

## Purpose

Many early startups face two common challenges:

1. Creating a **professional-looking website**
2. Avoiding backend costs during the early phase  

FormBridge solves this by:

- Keeping the UI fully customizable  
- Using Google Forms as a free and reliable data collection system  

---

## How It Works

1. User fills out a form on your website  
2. Input fields are mapped to corresponding Google Form fields  
3. Data is submitted using the Google Form’s hidden POST endpoint  
4. Responses are stored directly in Google Forms  

---

## Features

- No backend required  
- Fully customizable frontend  
- Zero cost data storage (via Google Forms)  
- Quick and easy integration  
- Google Form stays hidden from users  

---

## Project Vision

This is the **first of two bridge-based projects**:

- FormBridge → Google Forms integration  
- Upcoming → Tally.so integration  

The goal is to help startups:

- Launch faster  
- Reduce costs  
- Maintain a professional UI  

---

## Use Cases

- Startup landing pages  
- MVPs (Minimum Viable Products)  
- Contact forms  
- Waitlist/signup systems  

---

## Limitations

- Limited validation compared to full backend systems  
- Dependent on Google Forms structure  
- Not ideal for complex workflows  
- does NOT properly support file/image uploads
---

## Future Improvements
